## repetit-mcp

**MCP stdio transport does not require a daemon.** Claude Code spawns `repetit mcp` fresh per session; the process stays alive for the session duration but is managed entirely by the host. No manual startup or port needed.

**Project-level MCP config** (`.claude/settings.json`):
```json
{
  "mcpServers": {
    "repetit": {
      "command": "bun",
      "args": ["run", "mcp"],
      "cwd": "/Users/eliot/ws-ps/ankimo"
    }
  }
}
```

Fix for machine-specific path: added `"mcp": "bun src/cli/main.ts mcp"` script to `package.json`, then reference `bun run mcp` + `cwd` in settings instead of hardcoding the source file path. The `cwd` is still absolute (machine-specific) but more stable.

Note: `.claude/settings.json` is committed to git (team-visible). For a truly portable config the package would need to be published so `bunx repetit mcp` works. For now it's marked testing-only.

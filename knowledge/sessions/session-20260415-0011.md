## repetit-architecture

Rebuilt ankimo from scratch as `repetit` — a headless spaced repetition service.

Design decisions locked:
- Domain: `(learner_id, item_uid) → FSRS state + rating history`
- Items store opaque JSON payload (server never parses)
- Multi-learner from day 1; learner_id on every state/review row
- Channel decides rating (1-4); server only schedules/persists
- Pattern mirrors `ir` CLI: subcommand CLI + `repetit mcp` for stdio MCP

Layer separation:
- `src/core/` — pure TS, no HTTP coupling; only I/O is bun:sqlite
- `src/cli/` — delegates to core/, outputs JSON (--pretty for humans)
- `src/mcp/` — stdio MCP server via @modelcontextprotocol/sdk@1.29.0
- `src/http/` — Bun.serve thin adapter, X-Learner-Id header

## repetit-schema

```sql
learners (id PK, display_name, settings JSON, created_at)
items (uid PK, tags JSON array, payload JSON opaque, created_at, updated_at)
learner_states (learner_id FK, item_uid FK, fsrs_state JSON, due_at, last_reviewed_at)
reviews (id, learner_id FK, item_uid FK, rating 1-4, elapsed_ms, stability, difficulty, state, reviewed_at)
learner_config (learner_id PK FK, daily_new_limit, daily_review_limit, target_retention, tz_offset_minutes, fsrs_weights JSON)
```

Key: `due_at` is denormalized from fsrs_state for indexed queries. `tags` is a JSON array; queried via `json_each(i.tags)`.

## repetit-cli

Commands:
```
repetit learners add|list|config
repetit items import|list|get
repetit next [--learner] [--tag] [--limit]
repetit rate <uid> again|hard|good|easy [--learner] [--ms]
repetit queue [--learner] [--tag]
repetit mcp
repetit serve [--port]
```

All output JSON to stdout; errors to stderr as `{"error":"..."}`, exit 1. `--pretty` for humans.
`--learner` defaults to `"default"`.

## repetit-mcp

`repetit mcp` starts stdio MCP server. Tools: get_next, submit_review, import_items, list_items, get_due_counts — mirror the CLI surface exactly.

